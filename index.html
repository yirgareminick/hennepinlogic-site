// system.architecture

How the System Works

The pipeline from inbound signal to captured lead. Each step is automated, logged, and configurable to your business rules.

01

Inbound Signal

System monitors your phone line, web forms, Google Business Profile, and Facebook. When a lead event fires, we capture it instantly.

event: "missed_call"
source: "phone"
timestamp: 2026-01-04
status: "processing"
02

Automated Response

Within 60 seconds, the lead receives a personalized SMS from your business number. Message templates are configured during setup.

send_sms({
  to: lead.phone,
  from: business.number,
  template: "initial_response",
  personalization: true
})
03

Data Capture

Conversational flow gathers the information you need: service type, property address, urgency level, preferred callback window.

captured_data: {
  service: "water_heater",
  address: "verified",
  urgency: "high",
  callback: "asap"
}
04

Escalation Logic

If urgency flags trigger (flooding, gas smell, no heat), the system immediately notifies your on-call technician via SMS and call.

if (lead.urgency === "emergency") {
  notify_oncall(tech.phone);
  escalation_status: "sent"
}
05

Follow-Up Sequence

If the lead doesn't respond, automated follow-ups trigger at 2h, 24h, and 48h intervals. Configurable based on your preferences.

sequence: [
  { delay: "2h", type: "sms" },
  { delay: "24h", type: "sms" },
  { delay: "48h", type: "sms" }
]
06

Reporting Dashboard

Weekly reports show response times, lead volume, conversion status, and recovered opportunities. Data exports available.

report: {
  period: "weekly",
  leads_captured: 47,
  avg_response: "34s",
  recovered: 12
}
